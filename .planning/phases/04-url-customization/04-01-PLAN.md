  <action>
Replace the inline `onClick` handler with a named function that:

1. Computes `newTheme` (toggle between "light" and "dark")
2. Calls `setTheme(newTheme)` (existing next-themes integration)
3. Syncs to URL via `history.replaceState`:
   - Create `new URL(window.location.href)` to preserve all existing params and hash
   - If `newTheme === "light"`: delete the `mode` param (clean URLs for default state, per user decision)
   - If `newTheme === "dark"`: set `mode` param to `"dark"`
   - Call `history.replaceState(null, "", url.toString())` -- NOT pushState (per user decision: theme changes are not navigation events)

Replace the onClick on the button (currently line 18):
```typescript
// Before:
onClick={() => setTheme(isDark ? "light" : "dark")}

// After:
onClick={() => {
  const newTheme = isDark ? "light" : "dark";
  setTheme(newTheme);
  const url = new URL(window.location.href);
  if (newTheme === "light") {
    url.searchParams.delete("mode");
  } else {
    url.searchParams.set("mode", newTheme);
  }
  history.replaceState(null, "", url.toString());
}}
```

Do NOT change any other part of ThemeToggle.tsx -- styling, SVGs, aria attributes, mounted guard all stay the same.
