<objective>
Scaffold the Next.js 16 project with Tailwind CSS v4 and TypeScript, then build the complete three-tier CSS design token system with both light and dark mode semantic token sets. Wire up next-themes for theme state management.

Purpose: Establishes the foundational architecture that every future component depends on. The token system enforces that theme switching is a CSS-only operation -- semantic tokens swap via `[data-theme]` selectors while all layout/structure CSS stays mode-agnostic. This is the architectural backbone of THEME-03 (shared layout, different tokens) and the prerequisite for THEME-05 (flash-free theme init).

Output: A running Next.js project where `document.documentElement.setAttribute('data-theme', 'dark')` in the browser console instantly switches all colors, shadows, and glow effects from light to dark mode.
</objective>


**TIER 2: SEMANTIC TOKENS** (light mode as `:root` default, dark mode via `[data-theme="dark"]`)

Light mode (`:root`):
- `--bg-primary: var(--color-white)` -- page background (pure white per user decision)
- `--bg-secondary: var(--color-gray-50)` -- alternating sections
- `--bg-surface: var(--color-white)` -- cards, panels
- `--bg-surface-elevated: var(--color-white)` -- elevated cards, dropdowns
- `--text-primary: var(--color-gray-900)` -- headings, primary text
- `--text-secondary: var(--color-gray-600)` -- body text, descriptions
- `--text-muted: var(--color-gray-400)` -- captions, metadata
- `--text-on-accent: var(--color-white)` -- text on accent-colored backgrounds
- `--accent-primary: var(--color-blue-600)` -- links, CTAs, interactive elements (per user decision)
- `--accent-hover: var(--color-blue-700)` -- hover state for accent
- `--border-default: var(--color-gray-200)` -- standard borders
- `--border-subtle: var(--color-gray-100)` -- light dividers
- `--shadow-sm: 0 1px 2px 0 rgb(0 0 0 / 0.05)` -- subtle elevation
- `--shadow-md: 0 4px 6px -1px rgb(0 0 0 / 0.07), 0 2px 4px -2px rgb(0 0 0 / 0.05)` -- card elevation (Linear-inspired refinement)
- `--shadow-lg: 0 10px 15px -3px rgb(0 0 0 / 0.08), 0 4px 6px -4px rgb(0 0 0 / 0.04)` -- modal/dropdown elevation
- `--glow-accent: none` -- no glow in light mode
- `--glow-subtle: none` -- no glow in light mode

Dark mode (`[data-theme="dark"]`):
- `--bg-primary: var(--color-black)` -- page background (true black #0a0a0f per user decision)
- `--bg-secondary: #0f0f1a` -- alternating sections (slightly lighter than black)
- `--bg-surface: #1a1a2e` -- cards, panels (dark navy)
- `--bg-surface-elevated: #1e1e35` -- elevated surfaces
- `--text-primary: var(--color-gray-100)` -- headings (light text on dark)
- `--text-secondary: var(--color-gray-300)` -- body text
- `--text-muted: var(--color-gray-500)` -- captions, metadata
- `--text-on-accent: var(--color-black)` -- text on accent-colored backgrounds
- `--accent-primary: var(--color-cyan-400)` -- links, CTAs (cyan per user decision)
- `--accent-hover: var(--color-cyan-500)` -- hover state
- `--border-default: rgba(255, 255, 255, 0.1)` -- semi-transparent white borders (sci-fi feel)
- `--border-subtle: rgba(255, 255, 255, 0.06)` -- very subtle borders (per UX skill guidance: 0.06)
- `--shadow-sm: 0 1px 2px 0 rgb(0 0 0 / 0.3)` -- dark mode uses deeper shadows
- `--shadow-md: 0 4px 6px -1px rgb(0 0 0 / 0.4)` -- deeper shadow
- `--shadow-lg: 0 10px 15px -3px rgb(0 0 0 / 0.5)` -- deep shadow
- `--glow-accent: 0 0 20px rgba(34, 211, 238, 0.15)` -- medium cyan glow (Vercel energy, not neon terminal)
- `--glow-subtle: 0 0 10px rgba(34, 211, 238, 0.08)` -- subtle glow for secondary elements

- Font weights: `--font-normal: 400`, `--font-semibold: 600` (per user decision: medium weight contrast)

**SPACING TOKENS** (in `:root`, shared across both themes):
- 8pt grid: `--space-1` (4px) through `--space-32` (128px)

- `--radius-md: 0.5rem` (8px)
- `--radius-lg: 0.75rem` (12px -- consistent medium border-radius per UX skill)

2. **Update `src/app/globals.css`** to import tokens and configure dark variant:

```css
@import "tailwindcss";
@import "../styles/tokens.css";

@custom-variant dark (&:where([data-theme=dark], [data-theme=dark] *));
```

The `@import "../styles/tokens.css"` MUST come after `@import "tailwindcss"` so that the `@theme` block is processed by Tailwind. The `@custom-variant dark` line overrides Tailwind's dark mode to use `data-theme` attribute instead of class or media query.

3. **WCAG AA validation:** The following text/background pairs MUST pass 4.5:1 contrast ratio. These were verified in research, but confirm the exact hex values produce correct results:
   - Light: --text-primary (#111827) on --bg-primary (#ffffff) = 17.4:1
   - Light: --text-secondary (#4b5563) on --bg-primary (#ffffff) = 7.3:1
   - Light: --text-muted (#9ca3af) on --bg-primary (#ffffff) = 3.0:1 (NOTE: this is below 4.5:1 -- muted text is only for non-essential decorative text, never for content users must read. Use --text-secondary for readable body text.)
   - Light: --text-on-accent (#ffffff) on --accent-primary (#2563eb) = 4.6:1
   - Dark: --text-primary (#f3f4f6) on --bg-primary (#0a0a0f) = 18.1:1
   - Dark: --text-secondary (#d1d5db) on --bg-primary (#0a0a0f) = 14.1:1
   - Dark: --text-primary (#f3f4f6) on --bg-surface (#1a1a2e) = 12.4:1
   - Dark: --text-on-accent (#0a0a0f) on --accent-primary (#22d3ee) = 12.3:1

All essential text pairs exceed 4.5:1. The --text-muted token is explicitly for decorative/non-essential content only -- document this with a CSS comment.

**IMPORTANT ANTI-PATTERN RULES (add as CSS comments at the top of tokens.css):**
- Components MUST only reference Tier 2 semantic tokens: `--bg-*`, `--text-*`, `--accent-*`, `--border-*`, `--shadow-*`, `--glow-*`
- Components MUST NEVER reference Tier 1 primitives: `--color-*`
- This ensures theme switching is purely a CSS operation
