  <files>src/app/layout.tsx, src/components/layout/Header.tsx, src/components/layout/PageContainer.tsx</files>
  <action>
Build the layout infrastructure and the flash-prevention mechanism:

1. **Add blocking inline script** to `src/app/layout.tsx`:
   - Place a `<script>` tag using `dangerouslySetInnerHTML` as the FIRST child of `<head>` (before any stylesheets or other scripts)
   - The script MUST be synchronous and blocking (no async/defer)
   - Script logic:
     ```
     (function() {
       try {
         var params = new URLSearchParams(window.location.search);
         var mode = params.get('mode');
         if (mode === 'dark' || mode === 'light') {
           document.documentElement.setAttribute('data-theme', mode);
           try { localStorage.setItem('theme', mode); } catch(e) {}
         }
       } catch(e) {}
     })();
     ```
   - This runs BEFORE next-themes hydrates, so when next-themes initializes it finds the correct value already in localStorage and on the DOM
   - The try/catch ensures it fails silently if URLSearchParams is unavailable (old browsers) or localStorage is blocked (incognito)

2. **Create `src/components/layout/Header.tsx`**:
   - Sticky positioning: `position: sticky; top: 0; z-index: 50`
   - Backdrop blur: `backdrop-filter: blur(12px)` (per UX skill guidance)
   - Background: `var(--bg-primary)` with ~80% opacity for the glass effect (use `rgba` or CSS `color-mix`)
     - Light mode approach: `background: rgba(255, 255, 255, 0.85)` -- since bg-primary is theme-dependent, use a semi-transparent approach that works with both modes. Use a CSS custom property: add `--bg-header: rgba(255, 255, 255, 0.85)` to light mode and `--bg-header: rgba(10, 10, 15, 0.85)` to dark mode in tokens.css. OR use `background: var(--bg-primary); opacity` approach. Choose whichever is cleanest.
   - Bottom border: `1px solid var(--border-subtle)` (the semi-transparent white in dark mode creates the sci-fi border light effect)
   - Height: `var(--header-height)` (64px)
   - Inner container: max-width `var(--max-width)`, centered with `margin: 0 auto`, horizontal padding `var(--space-container)`
   - Flex layout with `justify-content: space-between; align-items: center`
   - Left placeholder div (for logo/name in Phase 2)
   - Right placeholder div (for nav + theme toggle in Phase 2/3)
   - Use semantic tokens ONLY -- no primitive `--color-*` references
   - Apply semantic tokens via Tailwind arbitrary values where possible (e.g., `bg-[var(--bg-primary)]`) or inline styles. Be consistent -- pick one approach for this project and use it throughout.

3. **Create `src/components/layout/PageContainer.tsx`**:
   - Max-width wrapper: `max-width: var(--max-width)` (1200px)
   - Centered: `margin: 0 auto`
   - Horizontal padding: `var(--space-container)` (fluid 16px -> 48px)
   - Full width on mobile, constrained on desktop
   - Accept `children` prop with proper TypeScript typing
   - Accept optional `className` prop for composition
   - Accept optional `as` prop for semantic HTML element (defaults to `div`) -- sections will use `<section>`, main content `<main>`

4. **Update `src/app/layout.tsx`** body structure:

   - The body wraps content in ThemeProvider (already from Plan 01)
   - Apply `line-height: var(--text-body-lh)` and `font-size: var(--text-body)` as base body styles
